O que é bug e debug? - JS

Bug é um problema que ocorre no código, muitas vezes por erro do programador, que impede o funcionamento correto do mesmo.

Debug é o ato de resolver os bugs encontrados no código ou também a forma que é feita a visualização de valores de variáveis ou fluxo do código.

Strict mode

Deixao JavaScript mais rigoroso na hora de programar, devendo ser declarado no topo do arquivo ou função.

Colocar o strict ajuda você a codificar de forma correta e não vai impedir ou limitar em nada o seu software ou program.

Veja um exemplo de uma variável declarada sem let/const/var.
EX:
"use strict"
opa = 'teste';

  • uncaught referenceErro: opa is not defined at window.onload(.(index):34)

Debugs com console.log

O meto bstante utilizado para debugs é o console.log, uma função que é utilizada apara mostrar os valores no console.

EX:
let a = 1;
let b = 2;
if(a == 1) {
a = b + 2;
}
console.log(a);
for(i = 0; i < b; i++) {
a = a + 2;
console.log(a);
}
if(a > 10) {
a == 25;
}

Debug com debugger

É uma funcionalidade que para o código quando atingir a linha debugger ou com erro. É bem mais eficiente e elegante visualmente.

EX:
let a = 1;
let b = 2;

if (a == 1) {
a = b + 2;
}
for (i = 0; i < b; i++) {
a = a + 2
}

debugger;

if (a > 10) {
a == 25;
}

Tratamento de input por função

Não podemos controlar os dados que o usuário enviar, então para o bom funcionamento do software, devemos tratar eles.

Vejamos um exemplo de tratamento de um number:

EX:
function checarNumero(valor) {
let resultado = Number(valor);
if(Number.isNaN(resultado)) {
return null;
} else {
return resultado;
}
}
console.log(checarNumero(5));
console.log(checarNumero('teste'));

Exceptions

Podemos criar erros no programa, caso alguma condição não seja atendida, porém as exceptions só geram o erro.

EX:
let a = 1;
if (a != 2) {
throw new Erro("O valor de a não pode ser 1")
}

  • uncaught Erro: O valor de a não pode ser 1 at window.onload(.(index):36)

Try and catch

O bloco try catch, vai tentar executar um código, caso não consiga, ele pode retornar o erro que esse código gerou, sendo assim muito útil para debug.

EX:

try {
let c = a + b;
} catch(erro) {
console.log("Algo deu errado: " + error);
}

Finally

Finally é executada independente do resultado do try/catch, podendo exixtir com try e catch ou apenas com try.

EX:

try {
let c = a + b;
} catch(error) {
console.log("Algo deu errado: " + error);
} finally {
console.log("Executor")
}

Assertion

Verificações de programas, que são utilizadas para assegurar que tudo ocorra da forma esperada.

EX:

function iterarArray(arr) {
if(arr.length == 0) {
throw new Error("O array precisa ter elementos");
} else {
for(let i = 0; i < arr.length; i++) {
console.log(arr[i]);
}
}
}
